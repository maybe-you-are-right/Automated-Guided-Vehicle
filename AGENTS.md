# AGENTS.md

This file provides guidance for AI coding agents working in this repository.

## 1. Purpose

This repository is a Renesas RA4M2-based infrared line-following robot project. The primary code under development is in `src/`; generated FSP code is in `ra_gen/`, `ra/`, and `Debug/ra_gen/`.

## 2. What to edit

Prefer editing handwritten source under:
- `src/hal_entry.c` - main entry and periodic control loop
- `src/app/APP_MOTOR.c` / `src/app/APP_MOTOR.h` - line-follow logic and differential speed control
- `src/algorithm/pid.c` / `src/algorithm/pid.h` - discrete PID controller
- `src/bsp/bsp_motor.c` / `src/bsp/bsp_motor.h` - motor driver abstraction
- `src/bsp/bsp_track.c` / `src/bsp/bsp_track.h` - line sensor interface

Avoid editing generated code unless the task is specifically about hardware initialization or FSP configuration:
- `ra_gen/`
- `Debug/ra_gen/`
- `ra/`
- `ra_cfg/`
- `configuration.xml`

Also avoid changing build artifact directories such as `Debug/` except for build and compile database generation.

## 3. Build commands

The build system is Makefile-based. Use the `Debug` directory for normal build operations:

```bash
cd Debug
make
make clean
make all
```

Important outputs:
- `game_project1.elf`
- `game_project1.srec`
- `game_project1.map`
- `game_project1.siz` (generated by `make all`)

## 4. Architecture overview

- `src/hal_entry.c`
  - main application entry
  - periodic 10ms task calling `app_motor_line_follow_update()`
  - default base speed control via macros
- `src/app/APP_MOTOR.c` / `APP_MOTOR.h`
  - line-follow PID integration
  - differential motor speed adjustment
  - manual control support
- `src/algorithm/pid.c` / `pid.h`
  - discrete PID algorithm
  - output limiting and integral clamping
- `src/bsp/bsp_motor.c` / `bsp_motor.h`
  - dual PWM per wheel (GPT2/GPT4): left P102/P103, right P301/P302
  - `motor_bsp_set_speed` uses complementary dual-PWM (100%, 100−|duty|)
- `src/bsp/bsp_track.c` / `bsp_track.h`
  - 8-channel mux: AD0/AD1/AD2 → P002/P001/P000, OUT → P013
  - 500 µs settle after address change; black line → OUT high → 1
- `src/app/app_uart.c` / `app_uart.h`
  - STOP/RUN commands; `HAL_TRACK_DEBUG_UART` (default 0) for periodic gray-scale print

## 5. Project conventions

- Comments and documentation are written in Chinese
- Naming convention: `layer_module_action()`
- Type names use `_t` suffix
- Compiler standard: C99 (project is C-based)
- Optimization: `-O2`
- Warning flags: `-Wall -Wextra`
- Floating-point ABI: `hard`

## 6. Common task guidance

- For PID tuning, adjust constants in `APP_MOTOR.c` and use `app_motor_last_line_error_get()` / `app_motor_last_pid_out_get()` for diagnostics.
- For line-follow speed changes, modify `HAL_LINE_FOLLOW_BASE_SPEED` or `HAL_LINE_FOLLOW_PERIOD_MS`.
- For motor/hardware fixes, update `bsp_motor.c` / `bsp_track.c` first.
- For generated peripheral init changes, update FSP config only when you understand `ra_cfg/`, `ra_gen/`, and `configuration.xml`.

## 7. Useful files

- `Debug/compile_commands.json` - compilation database for code navigation and analysis tools
- `script/fsp.ld` - linker script
- `ra_cfg/` - FSP configuration headers
- `ra_gen/` - generated initialization code

## 8. Notes for AI agents

- Preserve existing Chinese comments and naming style when editing
- Keep changes localized to `src/` unless the issue explicitly requires FSP-generated code or build system updates
- Prefer concise fixes over broad refactors in firmware code
