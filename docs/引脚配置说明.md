# 引脚配置说明

本文档整理 **RA4M2** 本工程中已由 FSP 配置、并在手写固件中使用的引脚与功能对应关系（在**原有说明结构**上按当前生成文件更新）。

- **引脚模式与复用**：以 **`ra_gen/pin_data.c`** 为准。  
- **GPT 通道与引脚绑定**：以 **`configuration.xml`** 中 `config.driver.gpt` 及 **`ra_gen/hal_data.c`** 为准。  
- **循迹 / 串口**：以 **`src/bsp/bsp_track.c`**、**`src/bsp/bsp_uart.c`** 为准。  
- **电机**：以 **`src/bsp/bsp_motor.c`** 为准；已与 **`g_timer3` / `g_timer4`** 双 GPT 双 PWM 对齐（见 §3）。

## 3. 电机驱动（双 PWM：两路 GPT，各 GTIOCA + GTIOCB）

### 3.1 FSP 已绑定的 GPT 引脚（与 `configuration.xml` 一致）

| GPT 硬件 | GTIOCA | GTIOCB | HAL 实例（当前 `hal_data`） |
|----------|--------|--------|----------------------------|
| GPT2 | **P103** | **P102** | **`g_timer3`**（`timer_cfg_t.channel = 2`） |
| GPT4 | **P302** | **P301** | **`g_timer4`**（`timer_cfg_t.channel = 4`） |

即：**P102、P103** 供左轮双 PWM（同一 **`g_timer3`**），**P301、P302** 供右轮双 PWM（同一 **`g_timer4`**）。**`bsp_motor.c`** 中正转侧用 **GTIOCA**，反转侧用 **GTIOCB**（与 `bsp_motor.h` 注释一致）。

### 3.2 与旧版四实例方案的差异

若日后在 FSP 中为左右轮各配 **两个独立 GPT 实例**（共四个 `g_timer*`），需在 **`bsp_motor.c`** 中改回为四路 `Open`/`DutyCycleSet` 映射；当前工程采用 **每轮一个 GPT、两脚 A/B**，更省定时器资源。

### 3.3 PWM 周期

**`g_timer3_cfg`** 与 **`g_timer4_cfg`** 的 **`period_counts`** 须在 FSP 中配置一致；**`motor_bsp_init()`** 内对二者做一致断言。具体数值与频率见 **`ra_gen/hal_data.c`** 中 “Actual period” 注释。

## 4. 串口（SCI9）

| 引脚 | 功能 | 说明 |
|------|------|------|
| **P109** | SCI9 TXD | 连接主机 / USB 转串口的 RX |
| **P110** | SCI9 RXD | 连接主机 / USB 转串口的 TX |

- FSP 实例：**`g_uart9`**（波特率、数据位等见 **`ra_gen/hal_data.c`** 中 **`g_uart9_cfg`**）。  
- 应用层：**`src/bsp/bsp_uart.c`**；使用前调用 **`uart_bsp_init()`**（通常在 **`g_hal_init()`** 之后）。

**易混点（与旧说明一致）**：**右轮 PWM 不在 P110**；P110 为 **SCI9 RXD**。

## 5. 循迹传感器（8 路复用）

### 5.1 地址线（输出）

| 引脚 | 功能 | 说明 |
|------|------|------|
| P000 | AD0 | 地址位 0 |
| P001 | AD1 | 地址位 1 |
| P002 | AD2 | 地址位 2 |

通过 3 线二进制地址选择 **8 路中的 1 路**。实现见 **`src/bsp/bsp_track.c`**（**`track_set_address`** 等）。

### 5.2 数据线（输入）

| 引脚 | 功能 | 说明 |
|------|------|------|
| P013 | OUT | 数字输入，已配置上拉；读取前地址切换延时见 **`bsp_track.h`** 中 **`TRACK_MUX_DELAY_US`**（默认 5 µs） |

实现：**`track_bsp_read_channel` / `track_bsp_read_all` / `track_bsp_read_mask`**。

## 7. 修改引脚时建议同步更新的位置

| 若变更内容 | 建议处理 |
|------------|----------|
| PWM / SCI / IRQ 等外设复用 | 在 **e² studio FSP** 中改 **Pins** 与 **Stacks**，重新生成 **`ra_gen/`、`ra_cfg/`** |
| 电机定时器与引脚 | 同步 **`src/bsp/bsp_motor.c`**（及 **`bsp_motor.h`** 注释），并与 **`hal_data.h`** 中 **`g_timer*`** 一致 |
| 循迹地址 / OUT | 修改 **`src/bsp/bsp_track.c`** 中 **`TRACK_PIN_*`** 宏 |
| 本文档 | 更新 **`docs/引脚配置说明.md`**，避免与 **`pin_data.c`** / 硬件不一致 |

---

## 8. 备注

- **P104、P402** 为 **IRQ 输入**，不是 GPT 输出；请勿再按旧版「P104 电机 PWM」理解。  
- **P102、P103、P301、P302** 为 **GPT 外设引脚**（双 PWM 常用四线）。  
- 串口若与 PC 仅接 TX/RX/GND，而 FSP 中 UART **硬件流控** 仍启用 RTS/CTS 时，可能出现通信异常；可在 FSP 中将流控改为 **None** 后重新生成工程。
